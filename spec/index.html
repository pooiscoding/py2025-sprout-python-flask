2025 資訊之芽 Python 語法班 Python Flask 小作業

簡介

在本次作業中，我們會進行簡單的網頁前後端開發，從中學習網頁、網際網路相關知識、練習進階 Python 語法、接觸 Python Flask 網頁框架。


在本次作業中，你將完成:

  1. 一個 Python Flask 後端程式，回應使用者的 HTTP Request
  2. 製作簡單的一頁式網站，主題不拘
  3. 在網頁中提供到訪人次資訊，用後端完成
  4. 提供一個簡單的轉址回應
  5. 其他額外的前後端設計 (選做)
  6. 部屬你的 Python Flask 後端程式，提供服務

完成作業後，你需要繳交以下內容:

  1. 將專案原始碼放到一個 GitHub Repo，繳交 GutHub Repo 的連結
  2. 若無法以 GitHub Repo 呈現，將原始碼打包成 zip 壓縮檔，繳交 Google 雲端硬碟分享連結，亦可接受
  3. 部屬好的網站連結
  4. 簡單的影片說明，介紹你完成的內容 (操作前端即可，不需講解程式碼)
  5. 若無法以影片呈現，書面報告亦可接受

預計需要 1~2 小時，完成基礎的網頁程式。
根據你選做額外項目的難度、你的熟悉程度，需要更多時間，完成你的網頁設計。
預計需要 1 小時，完成所有的繳交工作。

一頁式網站

score: 20 points

任務說明:

在本項任務中，你需要達成以下目標:

在 browser 存取 URL: http://[server]/ 時，可以成功看到同學的網頁。

也就是，向 [server] 發送 HTTP GET Request ，存取 route 為 / (root) 時，會拿到同學的網頁。

一頁式網站說明:

網站 (Website): 網站是一個完整的、可以訪問的網路資源，通常有一個首頁作為入口，並可以通向許多其他的網頁。

網頁 (Web page): 網頁是網站中的一個單一頁面，是用 HTML 等技術編寫出來，能顯示在瀏覽器中的單一內容頁面。


一頁式網站，強調使用單一網頁，來呈現完整的內容，不使用多份網頁來呈現你的內容。

旨在說明不需要製作太大型的應用 (如: Blog 網站、含登入系統的網站等)，只要在單一頁面內呈現內容就好。


舉例而言:

  • 自我介紹網頁 範例
  • 一個簡單的推坑網頁 範例
  • 不知所云的網頁 範例
  • 一個簡單的 Hello World 也是可以啦

在此任務中，只要有最基礎的回覆 (例如 Hello World) 就可以通過。

在本網頁中的額外設計，會被算在 其他選做 的 額外前端設計 項目中。

內容不可腥羶色，不可違反社會善良風俗，包含但不限於: 賭博、詐騙、釣魚網頁等等，違者視情節輕重有相對應的處分。

可愛的貓咪圖片、J-POP 推坑，非常歡迎 !

測試方式:

1. 在瀏覽器中，存取 URL: http://[server]/ ，檢查回覆。

2. 在命令提示字元中，利用 curl 工具發送 HTTP GET Request，檢查回覆。

curl -v http://[server]/

任一測試檢查到正確回覆即可通過測試，拿到 20 分。

造訪人次統計

score: 30 points

任務說明:

在本項任務中，你需要達成以下目標:

累計造訪人次說明:

你必須在後端回應 HTML 文件時，動態地將累計人數替換進 HTML 檔中，進行回應。

具體而言，你會需要用到 render_template() 幫助你完成任務。

此外，本任務要求永久性保存造訪人次。也就是，即使 Python Flask 程式重啟，依然能記得過去累計的造訪人次。

具體而言，你可以將累計造訪人次儲存在本地檔案中，透過 FILE I/O 讀寫檔案，來紀錄、更新造訪人次。

測試方式:

九次的存取都檢查到正確的造訪人次，並確認同學的實作正確，即可通過測試，拿到 30 分。

Download Files

score: 10 points

任務說明:

在本地端新增一個資料夾 download ，存放所有可以被下載的檔案。

實作一個 route /download ，使用瀏覽器存取 URL: http://[server]/download 時，會回傳包含若干個檔名的網頁，表示 download 資料夾中，可以被下載的檔案們。

實作 route /download/[file]，使用瀏覽器存取 URL: http://[server]/download/[file] 時，會下載該檔案。

檔案內容皆為: test123。

當檔案不存在時，回應 404 Not Found。

目錄說明:

請注意檔案權限相關問題。

你可以使用以下程式碼獲取 download 目錄內的所有 entry 名，包含檔案名、資料夾名:

import os
# get a list of file/directory names in download directory
files = os.listdir('download')
# generate your response with the files list

你可以選擇不要在 download 資料夾中，放置其他資料夾，來避免奇怪的問題。

你也可以挑戰自己，實作 download 資料夾中，可以包含其他資料夾的程式。

下載檔案回應說明:

請注意檔案權限相關問題。

請注意惡意的檔案路徑問題。

推薦你使用以下程式碼回傳檔案:

return send_from_directory('download', [FILENAME], as_attachment=True)

其中 as_attachment 參數為 True 時，回傳的 HTTP Response 會在 header 中放入 Content-Disposition: attachment 的標記。

瀏覽器收到帶有此參數的 Response 時，會將回應的檔案解釋成附件，將檔案下載到本地端。(而不是將檔案內容內嵌在瀏覽器的畫面中)

測試方式:

1. 使用瀏覽器，存取 URL: http://[server]/download，檢查回覆。

2. 隨機選一個顯示在 /download 網頁中的檔名，在命令提示字元中，利用 curl 工具發送 HTTP GET Request，檢查回覆。

curl -v http://[server]/download/[filename]

3. 隨機選一個沒有顯示在 /download 網頁中的英文字串(不超過 10 個字母)，在命令提示字元中，利用 curl 工具發送 HTTP GET Request，檢查回覆。

curl -v http://[server]/download/[filename]

4. 在命令提示字元中，利用 curl 工具發送 HTTP GET Request，檢查存取以下 URL 的回覆 (應為 404 Not Found)。

curl -v http://[server]/download/../app.py

通過第一項測試，可得 5 分。

在通過第一項測試的基礎下，通過第二項測試，可得 3 分。

在通過第一項測試的基礎下，通過第三項測試，可得 1 分。

在通過第一項測試的基礎下，通過第四項測試，可得 1 分。

已讀

score: 10 points

任務說明:

請實作一個 route /get_tracker，在瀏覽器存取 URL: http://[server]/get_tracker 後，會獲得一個 admin: [id] 的 cookie。

[id] 為 server 提供的數字，用來唯一辨識一個 tracker (追蹤器)。每次存取 /get_tracker 時，需獲得不同的 id。

請實作一個 route /tracker?id=[id]，被存取時會回傳一個 1px*1px 的透明圖片。

若存取者帶有 admin: [id] 的 cookie，則不會更新此 tracker 的狀態。

若存取者不帶有 admin: [id] 的 cookie，則將 id 為 [id] 的 tracker 狀態更新成已讀。

最後實作一個 route /read?id=[id] 的網頁，顯示 id 為 [id] 的 tracker 是否為已讀狀態。

若已讀，則顯示 Yes；若未讀，則顯示 No。
